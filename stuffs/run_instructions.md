# Builder App - Configuration and Run Guide

This document outlines everything you need to know to run the `BuilderApp` codebase on Android, iOS, and real devices.

Since we generated the base native iOS and Android folders using `npx expo prebuild`, you have a "bare" React Native project managed by Expo. 

## 1. Running on an Android Emulator

The error you experienced earlier (`could not connect to TCP port 5554`) means the Android emulator is either not running or not responding properly.

### Steps to Fix and Run:
1. **Open Android Studio**.
2. Go to **Device Manager** (or Virtual Device Manager).
3. **Start an Android Virtual Device (AVD)** (e.g., Pixel 7) by clicking the Play button.
4. Wait for the emulator to fully boot up to the home screen.
5. In your VS Code terminal, make sure you are in the `BuilderApp` directory:
   ```bash
   cd d:\projects\demo\AI\BuilderApp
   ```
6. Run the Android build command:
   ```bash
   npx expo run:android
   ```
   *This will compile the `android` folder into an APK and install it on the running emulator.*

## 2. Running on iOS

Because you are using a **Windows** machine, you cannot locally compile the iOS base code inside the `ios/` folder (which requires macOS and Xcode). However, you have two great alternatives with Expo:

### Option A: Use Expo Go (Recommended for Windows)
If you don't need highly custom native iOS modules right now, the easiest way to see the iOS version is via Expo Go.

1. Ensure you are in the `BuilderApp` folder.
2. Run the Expo development server:
   ```bash
   npx expo start
   ```
3. A QR code will appear in your terminal.
4. Download the **Expo Go** app from the App Store on your physical iPhone or iPad.
5. Open the **Camera** app on your iPhone and scan the QR code. It will prompt you to open Expo Go and load your app instantly.

### Option B: Build it in the Cloud (EAS Build)
If you require customizing the iOS native code but only have Windows, you can use Expo Application Services (EAS) to build it in the cloud.
1. Run `npm install -g eas-cli`
2. Run `eas login`
3. Run `eas build --platform ios`
This will generate a `.tar.gz` with a simulator build or an actual `.ipa` file if you have an Apple Developer account.

## 3. Running on Real Physical Devices

### Real Android Device
1. **Enable Developer Options on your phone**:
   - Go to Settings > About Phone.
   - Tap "Build Number" 7 times until it says "You are a developer".
2. **Enable USB Debugging**:
   - Go to Settings > System > Developer Options.
   - Turn on **USB Debugging**.
3. Plug your phone into your Windows PC via USB.
4. (Optional) Run `adb devices` in your terminal to verify your computer sees the phone. You should see a device ID followed by the word `device`.
5. Run the build command in the `BuilderApp` directory:
   ```bash
   npx expo run:android
   ```
   *This will compile the app and install it directly via USB onto your physical phone.*

### Real iOS Device
As mentioned in Option A above, the easiest way to run the development build on a physical iPhone while using Windows is to use the **Expo Go** app and scan the QR code generated by `npx expo start`.

## Troubleshooting Checklist
- **`ENOENT: no such file package.json`**: Always double check you are inside `BuilderApp` and not the root `d:\projects\demo\AI` folder.
- **`ADB Server port` errors**: Sometimes the ADB server gets stuck. You can kill it manually by running `adb kill-server` and then `adb start-server` in the terminal before running your project.
